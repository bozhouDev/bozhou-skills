# Viral Ad Writer

## Overview

一个基于真实爆款案例的商单推文生成系统。核心理念：**不凭空生成广告，而是从真实爆款数据中学习写作模式**。

## When to Use

**使用场景：**
- 收到品牌方 brief，需要撰写商单推文
- 产品类型：AI 工具、App 应用、在线课程
- 目标平台：Twitter/X、小红书、微博等社交媒体
- 需要爆款感，而非传统广告腔

**不使用场景：**
- 正式的品牌声明或公告
- 长篇深度评测文章
- B2B 白皮书或技术文档

## Core Workflow

```
1. 接收 Brief
   ↓
2. 分析产品类型 → 选择对应案例库
   ↓
3. 读取案例（从中学习模式）
   ↓
4. 生成推文（默认 1 版本，用户要求时再加）
   ↓
5. 自检（对照禁止清单）
```

## Quick Reference

### 写作核心原则

| 维度 | 要求 | 示例 |
|------|------|------|
| **钩子** | 前 10 字必须抓眼球 | 数据冲击、反常识、痛点共鸣 |
| **结构** | 钩子 → 价值点 → CTA | 不超过 280 字（Twitter） |
| **语气** | 口语化、真实感 | "卧槽"比"令人惊叹"更有效 |
| **CTA** | 自然嵌入，非强推销 | "评论区要链接"vs"立即购买" |

| 产品类型 | 读取文件 | 何时使用 |
|----------|----------|----------|

## Implementation

### Step 1: 分析 Brief

从 brief 中提取：
- 产品类型（AI 工具/App/课程）
- 核心卖点（最多 3 个）
- 目标受众
- 任何必须提及的信息（活动时间、折扣码等）

### Step 2: 加载案例

```markdown
# 工作流程
1. **必读**：先读 `references/core-examples.md`（精选案例）
2. **按需读**：根据产品类型，读对应的分类案例文件
3. **可选**：案例不足时，参考 `references/templates.md` 的结构
```

### Step 3: 生成推文

**默认输出 1 个版本**，选择最适合该产品的策略：

| 策略 | 何时使用 |
|------|----------|
| **数据冲击型** | 产品有明确效率提升数据（时间/成本对比） |
| **痛点共鸣型** | 产品解决明确痛点问题 |
| **场景化叙事型** | 产品需要体验感/代入感 |

**用户要求多版本时**，再输出 2-3 个不同策略的版本供选择。

### Step 4: 自检清单

生成后，对照以下清单检查：

**✅ 必须做到：**
- [ ] 前 10 字能独立成为吸引点
- [ ] 包含至少 1 个具体数字/案例
- [ ] 语气口语化，无广告腔
- [ ] CTA 自然，非强推销

**❌ 严禁出现：**
- [ ] "震惊""颠覆""革命性"等夸张词
- [ ] "强烈推荐""不容错过"等硬推销
- [ ] 超过 3 个感叹号
- [ ] 堆砌形容词（"强大的、智能的、高效的"）

## Common Mistakes

### ❌ 错误 1：忽略案例，凭空生成

**表现：**
```
震惊！这款 AI 工具彻底改变了我的工作方式！
强大的功能，智能的算法，不容错过！
```

**问题：** 完全是广告腔，没有参考任何真实爆款模式

**解决：** 必须先读案例，找到相似的成功模式再写

---

### ❌ 错误 2：案例照搬，没有改编

**表现：** 直接复制案例原文，只替换产品名

**问题：** 失去原创性，可能触发抄袭问题

**解决：** 提取案例的**结构和策略**，而非原文

---

### ❌ 错误 3：生成后不自检

**表现：** 输出后立即提交，未对照禁止清单

**问题：** 可能包含广告腔、夸张词等低质量元素

**解决：** 生成后默认执行自检，确保符合禁止清单要求

---

### ❌ 错误 4：忽略产品类型，用错案例

**表现：** AI 工具产品用了课程类的案例模式

**问题：** 调性不匹配，转化率低

**解决：** 严格按照产品类型选择对应的案例库

## Output Format

### 默认格式（1 版本）

```markdown
## 📋 Brief 分析
- 产品类型：[AI工具/App/课程]
- 核心卖点：[最多3个]
- 参考案例：[使用了哪些案例编号]

---

## 📝 推文

[推文内容]

**策略：** [策略名称] — [为什么用这个策略]
**参考案例：** [具体案例编号]
```

### 多版本格式（用户要求时）

在默认格式基础上，增加：

```markdown
---

## 📝 备选版本 1：[策略名称]

[推文内容]

**策略说明：** [...]

---

## 📝 备选版本 2：[策略名称]

[推文内容]

**策略说明：** [...]
```

## Real-World Impact

**预期效果：**
- 减少 80% 的"广告腔"修改反馈
- 提升 3-5 倍的推文互动率
- 建立可持续的爆款学习库

**持续优化：**
每当发现新的爆款推文，添加到对应的案例库中，让系统持续学习。
