# 🚀 Viral Ad Writer Skill 使用指南

这是一个基于真实爆款案例的商单推文生成系统。通过学习成功案例，而非凭空生成广告，帮你写出有爆款潜力的商单内容。

---

## 📁 文件结构

```
viral-ad-writer/
├── README.md                      # 本文件：使用指南
├── SKILL.md                       # 主文件：写作流程和核心原则
└── references/                    # 案例和模板库
    ├── core-examples.md           # ⭐ 精选10条完整拆解案例（必读）
    ├── templates.md               # 6个可复用模板
    ├── examples-ai-tools.md       # AI工具类案例（按需）
    ├── examples-apps.md           # App应用类案例（按需）
    └── examples-courses.md        # 在线课程类案例（按需）
```

## ⚡ 快速开始

### 第一步：填充案例库

在使用 Skill 之前，你需要先填充案例库：

1. **必填**：`references/core-examples.md`
   - 填入 10 条你收集的爆款推文
   - 每条都要完整拆解（原文、数据、分析、可复用点）
   - 这是核心素材，必须完成

2. **选填**：分类案例文件
   - 根据你接的商单类型，填充对应文件
   - 如果主要接 AI 工具商单，优先填 `examples-ai-tools.md`
   - 每个文件至少填 3 条案例

3. **自动生成**：`references/templates.md`
   - 填完案例后，可以让 AI 帮你从案例中提取模板
   - 或者直接使用现有的 6 个通用模板

### 第二步：使用 Skill 生成推文

在 Claude Code 中调用 Skill：

```bash
# 直接在对话中说：
我收到一个商单 brief：
[粘贴品牌方的 brief]

请用 viral-ad-writer skill 帮我生成 3 个版本的推文
```

AI 会自动：
1. 分析产品类型
2. 读取对应的案例文件
3. 参考案例和模板生成 3 个版本
4. 进行自检（对照禁止清单）

```
**为什么爆了：**
1. 数据对比极端（3小时 → 10分钟），制造认知冲击
2. 功能拆解具体（列大纲、写内容、配图），可操作性强
3. 口语化表达（"活儿""搞定""最爽"），真实感强

**关键元素：**
- ✅ 具体数字对比（18 倍效率提升）
- ✅ 功能点用 bullet points 展示（清晰易读）
- ✅ CTA 自然（扣 1 私发，低压力）

**模板提取：**
```

```

**适用场景：**
- 适合有明确效率提升数据的 AI 工具
- 适合功能较多需要拆解说明的产品

- "评论区扣 1"
```

### 快速填写技巧

1. **收集爆款推文**
   - Twitter/X 搜索：`AI工具 min_faves:1000`
   - 小红书搜索：关键词 + 按赞数排序
   - 微博搜索：话题 + 热门

2. **数据记录**
   - 点赞、转发、评论数要准确
   - 记录发布时间（分析时效性）

3. **拆解要点**
   - 前 10 字单独拎出来（钩子分析）
   - 用 bullet points 列功能点
   - 找到"为什么爆"的核心原因（1-3 个）

4. **可复用点提取**
   - 用占位符（如 `{产品}` `{任务}`）表示可替换部分
   - 记录话术库（可直接套用的金句）

## 🧪 测试建议

### 首次使用测试

填完案例库后，用一个真实的 brief 测试：

1. **准备测试 brief**
   ```
   产品：[具体产品名]
   类型：[AI工具/App/课程]
   核心卖点：[列出2-3个]
   目标用户：[描述]
   平台：[Twitter/小红书/微博]
   字数限制：[如280字]
   ```

2. **生成推文**
   - 调用 Skill 生成 3 个版本
   - 检查是否参考了对应的案例文件
   - 检查是否通过了自检清单

3. **自我评估**
   - [ ] 钩子（前 10 字）能否独立吸引人？
   - [ ] 是否有具体数字/案例？
   - [ ] 语气是否口语化？
   - [ ] CTA 是否自然？
   - [ ] 有没有广告腔（"震惊""革命性"等）？

4. **实战验证**
   - 选择最好的版本发布
   - 记录实际互动数据（点赞/转发/评论）
   - 对比预期，分析原因

### 持续优化

每接完一个商单：
1. 记录推文的实际表现
2. 如果表现好（高于平均互动），添加到案例库
3. 如果表现差，分析原因，更新禁止清单

## 🎯 使用技巧

### 技巧 #1：案例多样化

确保案例库包含不同风格：
- **数据型**：有明确数字对比的
- **情感型**：强痛点共鸣的
- **故事型**：场景化叙事的
- **惊喜型**：反常识/超预期的

### 技巧 #2：定期更新案例

**每月更新计划：**
- 新增 2-3 条最新爆款案例
- 淘汰表现最差的旧案例（如果数据过时）
- 更新模板库（基于新案例优化）

### 技巧 #3：建立话术库

从案例中提取可直接复用的金句：
- "卧槽，太香了"
- "评论区扣 1"
- "刚用完就..."
- "没想到居然能..."

### 技巧 #4：A/B 测试

生成 3 个版本后：
- 选 2 个版本分别发布（不同时间/平台）
- 对比实际数据
- 记录哪种策略更有效

## 🚫 常见错误

### 错误 #1：案例太少

**问题：** 只填了 2-3 个案例就开始用

**影响：** AI 学习样本不足，生成的内容容易趋同

**解决：** 至少填满 `core-examples.md` 的 10 个案例槽位

### 错误 #2：案例拆解不深入

**问题：** 只复制原文，没有分析"为什么爆"

**影响：** AI 无法学习到真正的成功模式

**解决：** 每个案例必须完成"爆款原因分析"和"可复用点"

### 错误 #3：忽略自检环节

**问题：** 生成后不检查，直接用

**影响：** 可能包含广告腔、夸张词等低质量元素

**解决：** 严格执行 SKILL.md 中的自检清单

### 错误 #4：照搬案例原文

**问题：** 直接复制案例的句子，只改产品名

**影响：** 失去原创性，可能被判定为抄袭

**解决：** 学习案例的**结构和策略**，而非原文

## 📊 效果评估

### 短期指标（发布后 24 小时）

- **互动率**：(点赞 + 转发 + 评论) / 粉丝数
- **转化率**：评论区询问数 / 总互动数
- **传播系数**：转发数 / 点赞数

### 长期指标（持续跟踪）

- **案例库质量**：新增案例的平均互动数
- **生成效率**：从 brief 到终稿的时间
- **命中率**：生成的推文实际表现高于平均的比例

## 🔧 高级用法

### 组合策略

**策略 #1：多平台适配**
- 同一个产品，针对 Twitter、小红书、微博生成不同版本
- 根据平台特性调整语气和格式

**策略 #2：系列推文**
- 不只生成单条推文，生成一个系列（3-5 条）
- 第一条吸引，后续条深挖

**策略 #3：与竞品对比**
- 在 brief 中提供竞品信息
- 生成隐性对比的推文（不直接点名）

### 自动化工作流

1. **Brief 模板化**
   - 建立标准 brief 模板
   - 每次填空即可，减少沟通成本

2. **批量生成**
   - 一次提供多个商单 brief
   - 批量生成，统一自检

3. **数据追踪**
   - 建立推文表现追踪表
   - 自动计算平均互动率
   - 识别高效模板

## 💡 最佳实践

### 接到新商单后的 SOP

1. **分析 brief**（5 分钟）
   - 产品类型是什么？
   - 核心卖点是什么？
   - 目标用户是谁？

2. **调用 Skill**（2 分钟）
   - 提供完整 brief
   - 生成 3 个版本

3. **人工优化**（10 分钟）
   - 选择最佳版本
   - 微调细节（语气、emoji、hashtag）
   - 对照自检清单

4. **客户确认**（5 分钟）
   - 提供 2-3 个版本给客户选择
   - 说明每个版本的策略差异

5. **发布追踪**（持续）
   - 记录实际互动数据
   - 对比预期分析原因
   - 更新案例库

**总耗时：** 约 20 分钟（vs 传统方式 1-2 小时）

## 🙋 FAQ

### Q1: 我需要填满所有案例文件吗？

**A:** 不需要。优先填 `core-examples.md`（必填 10 条），然后根据你的商单类型填对应的分类文件（至少 3 条）。

### Q2: 案例越多越好吗?

**A:** 不是。质量 > 数量。10 条高质量案例 > 50 条普通案例。选择真正爆了的（点赞 > 1000），且拆解深入的案例。

### Q3: 如果我的产品类型不在分类中怎么办？

**A:** 在 `core-examples.md` 中填入相关案例即可。或者创建新的分类文件（如 `examples-saas.md`），并在 SKILL.md 中更新文件使用指引。

### Q4: 生成的推文太像广告怎么办？

**A:** 检查你的案例库，确保案例本身不是广告腔。如果案例质量没问题，可能是自检环节没做好，严格执行禁止清单。

### Q5: 可以用这个 Skill 写其他内容吗（非商单）？

**A:** 可以。这个 Skill 的核心是"从爆款学习"，不只适用于商单。你可以：
- 写个人品牌推文（添加个人 IP 案例）
- 写产品发布推文（添加产品 launch 案例）
- 写观点讨论推文（添加观点类爆款案例）

只要案例库对应，就能生成相应类型的内容。
