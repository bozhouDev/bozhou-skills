# Viral Case Curator

## Overview

一个智能化的推文案例分析和管理系统。核心功能：
1. 深度拆解爆款推文的结构、技巧和心理策略
2. 生成学习版（Part A）和机器可读版（Part B）
3. 智能评估案例价值，自动管理精选案例库

## When to Use

**使用场景：**
- 发现了一条爆款推文，想要深度学习其写作模式
- 需要收录新案例到 viral-ad-writer 的案例库
- 想要系统化分析为什么某条推文会爆
- 需要维护和更新精选案例库（如core-examples.md中的10条）

**输入内容：**
- 推文原文（必需）
- 数据表现：点赞、转发、评论、浏览等（可选但推荐）
- 产品类型（可选，会自动推断）

## Core Workflow

```
1. 接收推文 + 数据
   ↓
2. 深度分析（结构/技巧/心理/传播机制）
   ↓
3. 生成 Part A（完整学习版）
   ↓
4. 生成 Part B（Skill 案例格式）
   ↓
5. 评估是否应加入精选库 + 推荐替换策略
```

## Implementation

### Step 1: 信息采集与初步分析

从用户输入中提取：
- 推文原文（完整内容）
- 数据表现（如有）
- 产品/主题类型（自动推断或用户指定）

初步判断：
- 内容类型（教程/观点/故事/推荐/带货）
- 目标受众
- 核心卖点

### Step 2: 生成 Part A - 完整学习版

按照以下结构进行深度分析：

#### 1. 核心信息提取
- 一句话总结
- 核心卖点/观点
- 目标受众
- 内容类型

#### 2. 结构拆解
创建表格，逐句分析：
- 原文
- 功能作用（钩子/承诺/证明/CTA等）
- 设计意图

总结整体结构模式（如：痛点→方案→证明→行动）

#### 3. 文案技巧分析

**钩子类型识别：**
- 痛点共鸣
- 反常识/颠覆认知
- 数据冲击
- 悬念/好奇心
- 身份认同
- 紧迫感
- 争议性观点

**用词特点：**
- 具体数字使用
- 感官词汇
- 情绪词
- 口语化程度（1-5分）
- 金句/记忆点提取

**节奏与格式：**
- 句子长短变化
- 分行/留白策略
- Emoji 使用
- 阅读流畅度（1-5分）

#### 4. 心理策略分析

识别触发的心理机制：
- 社会认同
- 稀缺性
- 权威背书
- 互惠心理
- 好奇心缺口
- 损失厌恶
- 身份投射
- 情绪共鸣

详细说明最主要的 2-3 个心理触发点如何运作。

#### 5. 传播动力分析

从以下维度分析为什么会爆：
- **为什么会停留？**（前2句做对了什么）
- **为什么会读完？**（内容吸引力）
- **为什么会点赞？**（情绪/认同触发）
- **为什么会转发？**（转发者的社交货币）
- **为什么会评论？**（讨论空间）

#### 6. 可复用模板

抽象出写作模板：
```
第1句：[钩子类型 + 句式结构]
第2句：[承诺/价值点 + 句式]
...
结尾：[CTA类型]
```

#### 7. 仿写练习

用同样结构，针对以下主题之一进行仿写：
- AI写作工具
- 效率App
- 在线课程

（选择最接近原推文产品类型的主题）

#### 8. 关键学习点总结

用 3 个要点总结最值得学习的地方，并评分：
- 可复制性评分：__/5分
- 评分理由

---

### Step 3: 生成 Part B - Skill 案例格式

将 Part A 的分析压缩成机器可读的结构化格式（控制在 400 字以内）：

```markdown
## 案例：[产品/主题名称]

**原文摘要：**
[保留前3句完整原文]
[后续内容用一句话概括]

**数据：** 👍 [点赞] | 🔄 [转发] | 💬 [评论] | 👁 [浏览]

**产品类型：** [AI工具/App/课程/其他]

**钩子类型：** [主要钩子] + [次要钩子]（如有）

**结构模式：** [部分1]→[部分2]→[部分3]→[部分4]

**核心技巧（3条以内）：**
1. [最关键的技巧，一句话说清]
2. [第二关键]
3. [第三关键]

**金句：**
- "[原文中最有记忆点的句子]"
- "[第二个金句]"（如有）

**可复用模板：**
第1句：[句式结构，用[]标注可替换部分]
第2句：[句式结构]
...
结尾：[CTA类型]

**适用场景：** [什么类型的产品/什么目标适合用这个模式]
```

### Step 4: 案例价值评估与收录建议

分析完成后，评估这个案例的价值：

#### 评分维度（每项 1-5 分）：
1. **可复制性**：结构是否清晰，能否迁移到其他产品
2. **教学价值**：是否展示了独特或高级的技巧
3. **数据表现**：互动数据是否足够优秀
4. **代表性**：是否代表某一类成功模式
5. **新颖性**：是否展示了现有案例库没有的策略

**总分计算：** 总分 / 5 = 综合得分

#### 收录建议：

**如果综合得分 ≥ 4.0：**
- ✅ 推荐加入精选案例库（core-examples.md）
- 说明理由：[为什么这个案例值得精选]
- 如果当前已有 10 条精选，推荐替换：[案例编号]，理由是：[为什么要替换这个]

**如果综合得分 3.0-3.9：**
- ✅ 推荐加���分类案例库（examples-ai-tools.md / examples-apps.md / examples-courses.md）
- 说明加入哪个分类及理由

**如果综合得分 < 3.0：**
- ⚠️ 不推荐收录，但分析仍有学习价值
- 说明理由：[哪些方面不够优秀]

### Step 5: 执行收录（可选）

如果用户确认收录，执行以下操作：

1. **读取目标案例文件**
   ```
   如果加入精选库：读 references/core-examples.md
   如果加入分类库：读对应的 examples-*.md
   ```

2. **如果需要替换旧案例：**
   - 删除指定的旧案例
   - 在合适位置插入新案例

3. **如果直接追加：**
   - 在文件末尾追加新案例

4. **更新案例编号**（如果有编号系统）

5. **生成更新报告：**
   ```markdown
   ✅ 已更新：references/core-examples.md
   - 新增：[案例名称]
   - 替换：[旧案例名称]（如有）
   - 当前案例总数：[数量]
   ```

## Common Mistakes

### ❌ 错误 1：分析过于表面

**表现：** 只描述"用了数字""有emoji"等表面特征

**问题：** 没有深入到为什么这些技巧有效

**解决：** 必须分析背后的心理机制和传播动力

---

### ❌ 错误 2：模板过于具体

**表现：** 可复用模板中包含过多原推文的具体内容

**问题：** 无法迁移到其他产品/场景

**解决：** 用 [方括号] 标注所有可替换部分，保持结构的抽象性

---

### ❌ 错误 3：评估主观化

**表现：** 凭感觉给分，没有依据

**问题：** 收录标准不一致

**解决：** 严格按照 5 个维度逐项评分，给出具体理由

---

### ❌ 错误 4：替换建议不合理

**表现：** 推荐替换的旧案例其实很优秀

**问题：** 损失了高价值案例

**解决：** 替换时必须说明：
1. 为什么新案例更好
2. 旧案例的哪些方面已被新案例覆盖
3. 是否有其他低分案例可替换

## Best Practices

### 1. 数据驱动的评估

优先考虑有明确数据支持的案例：
- 点赞 > 1000（或所在平台的高标准）
- 转发率 > 5%
- 评论活跃度高

### 2. 多样性原则

在精选案例库中保持：
- 不同产品类型的平衡
- 不同钩子类型的覆盖
- 不同结构模式的展示

避免所有案例都是同一种套路。

### 3. 时效性考虑

- 优先收录近 6 个月的案例
- 如果案例超过 1 年，需评估其策略是否仍有效
- 定期审查案例库，淘汰过时的案例

### 4. 学习价值优先

收录标准不仅看数据，更看教学价值：
- 一个展示独特技巧的中等爆款 > 一个套路化的超级爆款
- 优先选择能"教会新东西"的案例

## Real-World Impact

**预期效果：**
- 建立科学的案例收录标准，避免主观判断
- 持续优化案例库质量，保持高价值密度
- 从每条爆款中提取可复用的写作模式
- 形成"分析→学习→收录→应用"的闭环

**持续迭代：**
- 每月审查一次精选案例库，淘汰低分案例
- 每季度分析案例库的覆盖面，补充缺失的类型
- 追踪收录案例的实际应用效果，验证其价值
