# Viral Case Curator

## Overview

从爆款推文中提取可复用模式，收录到案例库。

**三种模式：**
- **快速模式**（默认）：评估 + 可直接收录的案例格式
- **深度模式**（`--deep`）：完整结构拆解 + 心理机制分析
- **效果反馈**（`--feedback`）：记录自产推文的发布效果，形成闭环

## When to Use

- 发现爆款推文，想判断是否值得收录
- 需要快速提取推文的可复用模式
- 深度学习某条推文的写作技巧（使用 `--deep`）
- **自己生成的推文发布后有数据反馈**（使用 `--feedback`）

## Core Workflow

```
快速模式（默认）           深度模式（--deep）         效果反馈（--feedback）
     │                         │                          │
     ▼                         ▼                          ▼
  3维度评估                 完整结构拆解               提取推文+数据
     │                         │                          │
     ▼                         ▼                          ▼
  收录建议                  心理机制分析                快速复盘
     │                         │                          │
     ▼                         ▼                          ▼
  案例格式输出               3维度评估              写入 my-posts.md
 （可直接粘贴）                  │
                               ▼
                          案例格式输出
```

## 快速模式（默认）

### Step 1: 信息提取

从输入中提取：
- 推文原文
- 数据（点赞/转发/评论/浏览）
- 产品类型（自动推断）

### Step 2: 快速分析

用一段话总结：
- 钩子类型是什么
- 核心结构模式
- 最值得学习的1-2个技巧

### Step 3: 三维度评估

| 维度 | 问题 | 评分 |
|------|------|------|
| **可迁移性** | 结构是否清晰，能否用于其他产品？ | /5 |
| **增量价值** | 是否提供现有案例库没有的技巧？ | /5 |
| **数据验证** | 互动数据是否证明"这确实有效"？ | /5 |

**综合得分** = 平均值

**收录标准：**
- ≥ 4.0：推荐加入 `core-examples.md`
- 3.0-3.9：推荐加入分类案例库
- < 3.0：不推荐收录

### Step 4: 输出案例格式

**格式必须与 `core-examples.md` 完全一致**，可直接复制粘贴：

```markdown
## 案例 #N：[产品名称]（[产品类型]）

**原文摘要：**
[保留前2-3句完整原文，这是钩子部分，必须完整]
[后续内容用方括号概括主要段落，保留关键数字和转折点]

**数据：** 👍 [点赞] | 🔄 [转发] | 💬 [评论] | 👁 [浏览] | ⭐ [收藏]

**产品类型：** [AI 工具/App/课程]（[细分类型]）

**钩子类型：** [主要钩子] + [次要钩子]

**结构模式：** [部分1] → [部分2] → [部分3] → [部分4] → [部分5]

**核心技巧：**
1. **[技巧名称，4-8字]**：[详细说明这个技巧是什么、为什么有效，用原文例子支撑，50-80字]
2. **[技巧名称]**：[详细说明，50-80字]
3. **[技巧名称]**：[详细说明，50-80字]

**金句：**
- "[原文中最有记忆点的句子]"
- "[第二个金句]"

**可复用模板：**
第1句：[句式结构，用[]标注可替换部分]（[功能说明]）
第2句：[句式结构]
...
结尾：[CTA类型]

**适用场景：** [什么类型的产品/什么目标适合用这个模式，列举3-4个具体场景]

---
```

### 快速模式输出示例

```markdown
# 案例评估

## 快速分析

## 三维度评估

| 维度 | 评分 | 理由 |
|------|------|------|
| 可迁移性 | 5/5 | 结构清晰，教程+变现模式可用于任何降本增效工具 |
| 增量价值 | 4/5 | "主动预设异议"技巧在现有案例中较少见 |
| 数据验证 | 5/5 | 1933赞、46万浏览，数据优秀 |

**综合得分：4.7/5** → 推荐加入 core-examples.md

## 可收录格式

[完整案例格式，可直接复制]

---

**下一步：** 回复「收录」执行写入，或「收录到 examples-ai-tools.md」指定目标文件
```

## 深度模式（--deep）

在快速模式基础上，增加以下分析：

### 完整结构拆解

创建表格，逐句分析：

| 原文 | 功能 | 设计意图 |
|------|------|----------|
| [第1句] | 钩子 | [为什么这样写] |
| [第2句] | 承诺 | [为什么这样写] |
| ... | ... | ... |

### 心理机制分析

识别并详细说明最主要的 2-3 个心理触发点：

**可能的心理机制：**
- 社会认同：别人都在用/赚到钱了
- 稀缺性：信息差、时间窗口
- 损失厌恶：不行动会错过什么
- 好奇心缺口：制造认知落差
- 身份投射：让读者想象自己成为某种人

**分析格式：**
```
**[心理机制名称]**
- 原文触发点："[具体句子]"
- 运作方式：[为什么这句话能触发这个心理机制]
- 效果：[读者会产生什么反应]
```

### 传播动力分析

回答这5个问题：
1. **为什么会停留？**（前2句做对了什么）
2. **为什么会读完？**（内容如何维持注意力）
3. **为什么会点赞？**（触发了什么情绪/认同）
4. **为什么会转发？**（转发者获得什么社交货币）
5. **为什么会评论？**（留下了什么讨论空间）

## 效果反馈模式（--feedback）

**用途：** 记录通过 viral-ad-writer 生成并发布的推文效果，形成学习闭环。

**为什么重要：** 自产推文 + 真实效果数据 = 最有价值的案例。它证明了"你的风格 + 这套方法论"的结合是有效的。

### Step 1: 信息提取

从用户输入中获取：
- 推文原文（或让用户粘贴）
- 发布数据（点赞/转发/评论/浏览）
- 发布平台和时间

### Step 2: 快速复盘

用 2-3 句话总结：
- 用了什么策略（数据冲击/痛点共鸣/场景叙事）
- 什么做得好（钩子有效/CTA 转化高）
- 什么可改进（如有）

### Step 3: 写入记录

**目标文件：** `viral-ad-writer/references/my-posts.md`

**格式：**

```markdown
---

## [日期]：[产品名称]

**平台：** [Twitter/X / 小红书 / 微博]

**推文：**
> [完整推文内容]

**数据：** 👍 [点赞] | 🔄 [转发] | 💬 [评论] | 👁 [浏览]

**策略：** [使用的策略名称]

**复盘：**
- ✅ 有效：[什么做得好]
- 💡 改进：[下次可优化的点]

---
```

### 效果反馈输出示例

```markdown
# 效果反馈记录

## 信息确认
- 产品：Medeo
- 平台：Twitter/X
- 数据：👍 523 | 🔄 89 | 💬 42 | 👁 8.2万

## 快速复盘

用了「数据冲击 + 教程」策略。钩子用"0元复刻几十万的广告"制造认知冲击，效果不错（8万浏览说明停留率高）。CTA 可以更自然一些，"试试吧"略显生硬。

## 已记录

✅ 已写入：viral-ad-writer/references/my-posts.md
当前自产案例数：1
```

### 何时触发效果反馈

用户可能的表述：
- "这条推文发了，数据是 xxx"
- "上次生成的推文效果不错，帮我记录一下"
- "反馈一下效果：点赞 500，转发 80"

识别到这类意图时，进入效果反馈模式。

## 收录执行

用户确认收录后：

1. **读取目标文件**
   - 精选库：`viral-ad-writer/references/core-examples.md`
   - 分类库：`viral-ad-writer/references/examples-[type].md`
   - 自产库：`viral-ad-writer/references/my-posts.md`（效果反馈模式专用）

2. **确定插入位置**
   - 如果需要替换，说明替换哪条及理由
   - 如果直接追加，添加到文件末尾

3. **执行写入**

4. **输出确认**
   ```
   ✅ 已更新：references/core-examples.md
   - 新增：[案例名称]
   - 替换：[旧案例名称]（如有）
   - 当前案例总数：[数量]
   ```

## Common Mistakes

### ❌ 技巧说明过于简略

**错误：**
```
**核心技巧：**
1. 用了数字对比
```

**正确：**
```
**核心技巧：**
1. **极端数字对比制造认知冲击**：用「几十万美金 vs 0元」「2-4周 vs 20分钟」的极端对比，让读者产生"这不可能"的好奇心，同时具体数字比"很贵""很快"更有说服力
```

### ❌ 可复用模板过于具体

**错误：**
```
第1句：太夸张了，小红书和闲鱼很快就会有这个生意了
```

**正确：**
```
第1句：太夸张了，[平台名]和[平台名]很快就会有这个生意了（预言钩子+紧迫感）
```

### ❌ 评估缺乏依据

**错误：**
```
增量价值：4/5
```

**正确：**
```
增量价值：4/5 | "主动预设异议回应竞品"技巧在现有10条精选案例中仅出现1次
```

## Best Practices

### 1. 数据门槛参考

- Twitter/X：点赞 > 500，转发率 > 5%
- 小红书：点赞 > 1000，收藏 > 500
- 如果没有数据，需在评估中说明"数据未知，仅基于内容评估"

### 2. 案例库多样性

精选库应覆盖：
- 不同产品类型（AI工具/App/课程）
- 不同钩子类型（数字冲击/痛点共鸣/反常识/故事）
- 不同结构模式（教程型/观点型/对比型）

### 3. 增量价值判断

收录前快速扫描现有案例库，确认新案例提供了：
- 新的钩子类型，或
- 新的结构模式，或
- 新的心理技巧

如果与现有案例高度相似，降低「增量价值」评分。
