# Viral Case Curator

深度拆解爆款推文，智能管理案例库的AI助手。

## 功能概览

这个skill是 `viral-ad-writer` 的配套工具，专门用于：

1. **深度分析爆款推文**：自动拆解结构、技巧、心理策略
2. **生成双版本输出**：学习版（Part A）+ 机器可读版（Part B）
3. **智能案例管理**：评估价值、推荐收录、自动替换旧案例

## 快速开始

### 基础用法

```bash
/viral-case-curator

# 然后粘贴你想分析的推文
```

### 完整输入格式

```markdown
【待分析推文】
[粘贴完整推文内容]

【数据表现】
点赞：1234 | 转发：567 | 评论：89 | 浏览：10000

【产品类型】（可选）
AI工具 / App / 课程
```

## 输出说明

### Part A：完整学习版

供你个人学习，包含：
- 逐句结构拆解
- 文案技巧分析（钩子类型、用词、节奏）
- 心理策略解析
- 传播动力分析
- 可复用模板
- 仿写练习
- 学习要点总结

### Part B：Skill 案例格式

可直接复制到案例库，格式如下：
```markdown
## 案例：[产品名]

**原文摘要：** [前3句 + 概括]
**数据：** 👍 1234 | 🔄 567 | 💬 89
**产品类型：** AI工具
**钩子类型：** 数据冲击 + 好奇心
**结构模式：** 钩子→价值点→证明→CTA
**核心技巧：**
1. 用具体数字制造反差
2. 结尾留悬念引导点击
**金句：**
- "花了3小时做的事，现在只要3分钟"
**可复用模板：**
第1句：[具体时间对比] + [效果数字]
第2句：[痛点描述]
...
**适用场景：** 效率类工具推广
```

### 案例价值评估

自动评分（1-5分）：
- 可复制性
- 教学价值
- 数据表现
- 代表性
- 新颖性

**综合得分 ≥ 4.0**：推荐加入精选库（core-examples.md）
**综合得分 3.0-3.9**：推荐加入分类库
**综合得分 < 3.0**：仅作学习参考

## 智能替换策略

当精选案例库已满（如10条）时，系统会：

1. 分析所有现有案例的价值
2. 找出最应该被替换的案例
3. 说明替换理由：
   - 旧案例哪些方面已过时
   - 新案例覆盖了哪些优势
   - 是否有策略重复

## 工作流程

```
发现爆款推文
     ↓
使用 /viral-case-curator 分析
     ↓
获得 Part A（学习） + Part B（收录）
     ↓
根据评分决定是否收录
     ↓
如收录：自动更新案例库
     ↓
virall-ad-writer 立即可用新案例
```

## 使用技巧

### 1. 批量分析

如果你收藏了多条爆款推文，可以依次分析：
```
/viral-case-curator
[推文1]

/viral-case-curator
[推文2]
...
```

### 2. 定期审查

建议每月运行一次案例库审查：
- 检查旧案例是否过时
- 评估案例覆盖面是否全面
- 淘汰低价值案例

### 3. 学习优先

不是所有分析都需要收录。有些推文虽然爆了，但策略不可复制，这种情况：
- Part A 仍有学习价值（了解为什么爆）
- Part B 可以不收录（因为无法复用）

## 常见问题

### Q: 什么样的推文值得分析？

A: 满足以下条件之一：
- 数据表现优秀（点赞/转发高）
- 使用了独特的写作技巧
- 代表某一类成功模式
- 你个人觉得很有启发

### Q: 分析后一定要收录吗？

A: 不一定。根据评分决定：
- ≥ 4.0 分：强烈推荐收录
- 3.0-3.9 分：可以收录
- < 3.0 分：学习即可，不必收录

### Q: 如何决定替换哪个旧案例？

A: 系统会自动推荐，依据：
1. 旧案例的评分
2. 策略是否重复
3. 时效性（超过1年的案例优先替换）
4. 教学价值（低价值的优先替换）

### Q: 可以手动修改分析结果吗？

A: 当然可以。Part B 的格式是标准化的，你可以：
- 调整技巧描述
- 优化模板抽象程度
- 修改适用场景

## 最佳实践

1. **及时分析**：看到好推文立即分析，不要攒着
2. **数据完整**：尽量提供完整的数据（点赞/转发/评论）
3. **多样性**：收录案例时注意保持类型多样性
4. **持续更新**：案例库不是一次性的，需要持续迭代
5. **实践验证**：收录的模板要实际用于写作，验证其效果

## 与 viral-ad-writer 的配合

```
viral-case-curator（案例管理）
         ↓ 输出
core-examples.md（案例库）
         ↓ 读取
viral-ad-writer（推文生成）
         ↓ 输出
爆款商单推文
```

这两个skill形成闭环：
- curator 负责**学习**爆款
- writer 负责**应用**爆款
