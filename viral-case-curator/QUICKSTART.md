# 快速使用指南

## 30秒快速开始

1. 发现一条爆款推文
2. 输入 `/viral-case-curator`
3. 粘贴推文 + 数据
4. 获得完整分析 + 收录建议

就这么简单。

---

## 输入格式（3种）

### 最简版（推文即可）
```
/viral-case-curator

[粘贴推文内容]
```

### 推荐版（含数据）
```
/viral-case-curator

【待分析推文】
[粘贴推文内容]

【数据表现】
点赞：1234 | 转发：567 | 评论：89
```

### 完整版（含产品类型）
```

【产品类型】
AI工具 / App / 在线课程
```

## 你会得到什么

### 1. Part A：深度学习版

一份完整的拆解报告，包括：
- 逐句结构分析（表格形式）
- 文案技巧解析（钩子/用词/节奏）
- 心理机制拆解（为什么会爆）
- 可复用模板（可直接套用）
- 仿写练习（学以致用）
- 3个核心要点总结

**用途**：个人学习，理解爆款背后的逻辑

---

### 2. Part B：机器可读版

一份结构化案例，格式如下：
```markdown
## 案例：[产品名]

**原文摘要：** [前3句+概括]
**数据：** 👍 1234 | 🔄 567 | 💬 89
**产品类型：** AI工具
**钩子类型：** 数据冲击
**结构模式：** 钩子→价值→CTA
**核心技巧：**
1. ...
2. ...
**金句：**
- "..."
**可复用模板：**
第1句：[...]
...
**适用场景：** ...
```

**用途**：直接复制到案例库，供 viral-ad-writer 使用

---

### 3. 收录建议

智能评分 + 收录建议：
```
可复制性：5/5
教学价值：5/5
数据表现：4/5
代表性：5/5
新颖性：4/5

综合得分：4.6/5

✅ 推荐加入精选案例库
如案例库已满，推荐替换：[案例X]
理由：...
```

**用途**：决策是否收录，以及替换哪个旧案例

    │        ↓

## 关键指标说明

### 可复制性（1-5分）
- **5分**：结构清晰，每个模块可拆解，套用到其他产品不需修改逻辑
- **3分**：部分可复用，但需要较多调整
- **1分**：高度定制化，难以迁移

### 教学价值（1-5分）
- **5分**：展示了独特或高级技巧，有明确的学习点
- **3分**：技巧常规，但运用得当
- **1分**：没有明显的技巧亮点

### 数据表现（1-5分）
- **5分**：点赞 > 2000 或转发率 > 15%
- **3分**：点赞 500-2000 或转发率 5-15%
- **1分**：点赞 < 500 或转发率 < 5%

（注：具体标准根据平台和账号体量调整）

### 代���性（1-5分）
- **5分**：代表某一类成功模式的标杆案例
- **3分**：是某个策略的良好示例
- **1分**：个例，难以归类

### 新颖性（1-5分）
- **5分**：展示了案例库中没有的新策略
- **3分**：是现有策略的新组合方式
- **1分**：完全套用已有模式

## 常见错误

### ❌ 只看数据不看技巧

**错误做法**：只要点赞高就收录

**正确做法**：综合评估5个维度，优先选择可复用的

**原因**：有些推文爆了是因为作者本身粉丝多，策略本身不一定好

---

### ❌ 案例库堆积不管理

**错误做法**：发现好的就往里加，从不删除

**正确做法**：定期审查，维持精选库在10条左右

**原因**：案例太多会降低学习效率，精选才有价值

---

### ❌ 分析后不实践

**错误做法**：收藏了一堆分析报告，从不用

**正确做法**：分析完立即用 viral-ad-writer 尝试应用

**原因**：实践才能检验理论，发现真正有用的技巧

## 进阶技巧

### 1. 建立分析日记

每周分析3-5条推文，坚持记录：
- 什么类型的钩子本周最有效？
- 哪些技巧重复出现？
- 有什么新趋势？

1个月后，你会看到清晰的规律。

### 2. 对比分析

同时分析两条推文：
- 同类产品，一个爆了一个没爆
- 找出关键差异

这比单独分析学到的更多。

### 3. 主题专项

每月选一个主题深挖：
- 本月只分析"钩子"
- 收集20个不同类型的钩子案例
- 建立钩子图鉴

下月换"CTA"，再下月换"结构"。
