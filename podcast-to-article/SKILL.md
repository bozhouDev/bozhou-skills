# 播客/视频转文章 Skill

将播客、视频、音频内容通过 NotebookLM 深度研究后，转化为一篇有血有肉的文章。

## 依赖

- **notebooklm** CLI（已安装且已登录）
- **humanizer-zh** skill（去 AI 味）
- 用户的写作风格 skill（可选，如 bozhou-writing-style）

## 工作流程总览

```
添加源 → 概览调研 → 提出4个方向 → 用户选择 → 深度调研(存档) → 写作 → 去AI味 → 生成标题
```

---

## Phase 1: 准备源材料

用户会给你一个 URL（YouTube、播客链接）或告诉你 NotebookLM 里已经有了。

**如果需要添加源：**

```bash
# 创建笔记本（或用户指定已有的）
notebooklm create "主题名" --json

# 设置上下文
notebooklm use <notebook_id>

# 添加源
notebooklm source add "https://youtube.com/..." --json

# 等源处理完毕
notebooklm source list --json  # 确认 status 为 ready
```

如果源还在 processing，等它变成 ready 再进入下一步。可以用 `notebooklm source wait <source_id>` 等待。

---

## Phase 2: 概览调研（1 轮）

向 NotebookLM 提一个全面的概览问题，快速摸清内容全貌。

```bash
notebooklm ask "Please give me a comprehensive overview: 1) Who are the key people and their backgrounds? 2) What is the main topic/theme? 3) What are the top 3-5 key takeaways? 4) What is the structure/flow of the conversation?" --json
```

从返回的 JSON 中提取 `conversation_id`，后续所有提问都用 `-c <conversation_id>` 保持对话连贯。

解析回答，提取核心信息：人物、主题、关键论点、对话结构。

---

## Phase 3: 提出 4 个写作方向

根据概览内容，提出 4 个不同的写作角度。每个方向要包含：

- **方向名称**（一句话概括切入点）
- **聚焦内容**（这个方向重点写什么）
- **目标读者**（什么人看了会觉得有用）

用 `AskUserQuestion` 让用户选择 1-2 个方向。

**提方向的原则：**
- 不要做"全面总结"型方向——那是读书笔记，不是文章
- 每个方向要有明确的切入点和情绪基调
- 可以从这些维度找角度：方法论、人物故事、反常识观点、踩坑经验、行业趋势
- 方向之间要有差异，不要只是换个说法

---

## Phase 4: 深度调研（4-6 轮）

根据用户选择的方向，设计 4-6 轮有针对性的提问。每轮聚焦一个子主题。

**提问策略（根据选定方向组合使用）：**

| 子主题 | 示例问题方向 |
|--------|-------------|
| 核心方法论 | 具体策略、流程、工具、数字 |
| 金句与观点 | 最有启发的原话、核心信念、反常识观点 |
| 踩坑与失败 | 具体困难、错误、教训、差点失败的故事 |
| 个人故事 | 关键转折点、情感细节、未公开说过的事 |
| 增长与商业 | 增长策略、变现模式、关键数字 |
| 给别人的建议 | 实操建议、框架、心态建议 |

每轮提问都用 `--json -c <conversation_id>` 保持上下文：

```bash
notebooklm ask "具体问题..." --json -c <conversation_id>
```

**关键：把每轮 Q&A 写入临时文档。** 在用户项目目录下创建一个调研笔记：

```
文件路径：当前工作目录/podcast-research-<日期>.md
```

文档格式：

```markdown
# 调研笔记：[内容标题]
日期：YYYY-MM-DD
来源：[URL]
选定方向：[用户选的方向]

## Round 1: [子主题]
**问题：** ...
**回答：** ...
**关键提取：** （3-5 个要点）

## Round 2: [子主题]
...
```

每轮调研完成后立即写入文档，不要攒到最后。

---

## Phase 5: 写作

### 5.1 设计开头钩子

开头是最重要的部分。在写全文之前，先单独构思 2-3 个不同风格的开头钩子：

**钩子类型参考：**
- **反差型**：用一个出人意料的事实或对比抓住注意力（例：计划了 ABCD 四条路，结果走了 Plan E）
- **场景型**：把读者拉进一个具体的场景和画面（例：他坐在 Joshua Tree 的石头上，三个小时没动）
- **问题型**：抛出一个让人想知道答案的问题（例：一个产品经理怎么做到 120 万订阅的？）
- **金句型**：用嘉宾说过的最有力的一句话开场

用 `AskUserQuestion` 让用户选一个开头方向，或者告诉你他想要哪种感觉。

### 5.2 撰写全文

基于调研笔记和选定的开头，撰写完整文章。

**写作要求：**
- 字数 2000-5000 字（根据内容丰富程度调整）
- 不是视频笔记或内容摘要——是用视频素材为原料的独立文章
- 要有主线（贯穿全文的核心论点或叙事弧线）
- 细节要具体：名字、数字、时间线、具体故事
- 引用嘉宾原话时用 **加粗** 标注

**如果用户有写作风格 skill（如 bozhou-writing-style）：** 加载该 skill 并严格遵循其风格要求。写之前先读一遍 skill 的核心规则和禁用清单。

**如果没有特定风格 skill：** 用清晰、自然、口语化的中文写作。避免商业黑话和套话。

### 5.3 确认文章保存位置

用 `AskUserQuestion` 问用户文章保存到哪里，或者根据上下文推断（比如用户之前的文章都放在某个目录）。

---

## Phase 6: 去 AI 味

文章写完后，调用 humanizer-zh skill 进行去 AI 味处理：

```
/humanizer-zh 对文件 [文章路径] 进行去AI味优化
```

如果用户有写作风格 skill，在调用 humanizer-zh 时一并提及，让它结合风格要求优化。

去 AI 味重点关注：
- 对称排比（三段式、四段式并列结构）
- "不是...而是..."过度使用
- 工整的金句/slogan（听起来像能印在 T 恤上的话）
- AI 高频词（恰恰、近乎、至关重要、值得注意的是）
- 段尾的总结升华（每段最后一句话都在"扣题"）
- 过度使用破折号做并列

---

## Phase 7: 生成标题

最后一步：为文章生成 3-5 个标题选项。

**好标题的特征：**
- 有信息量——看完标题就知道文章大概讲什么
- 有钩子——让人想点进去看
- 不是标题党——承诺和内容匹配
- 长度 15-30 字

**标题公式参考：**
- [人名]：[具体事件/数字]，[一个启发]
- [反常识结论]：[一个人怎么做到的]
- [具体数字/事实] + [背后的故事/方法]

用 `AskUserQuestion` 给出 3-5 个标题让用户选择，或者用户可以自己改。选定后更新文章标题。

---

## 检查清单

文章完成后自查：

- [ ] 开头是否有钩子（3 秒内抓住注意力）
- [ ] 是否有贯穿全文的主线（不是罗列要点）
- [ ] 细节是否具体（有名字、数字、场景）
- [ ] 嘉宾原话是否用加粗标注
- [ ] 是否已去 AI 味
- [ ] 标题是否吸引人且诚实
- [ ] 字数是否在 2000-5000 之间
- [ ] 调研笔记是否已保存
